# Context Step 1: FHIR Patient Data Creation

## 1. Objective
The primary objective of Step 1 is to simulate patient clinical records for a typical regional hospital in Jakarta, Indonesia (*RSUD Tarakan Jakarta*). This data acts as the raw EHR source format before undergoing standardized research database transformation.

---

## 2. Demographic & Clinical Profile Design

### 2.1 Disease Cohorts
To replicate actual clinical volumes, the simulator generates a balanced population combining both **communicable** and **non-communicable** (chronic) diseases:
* **Communicable Diseases:**
  * **Tuberculosis (TB):** Standard Indonesian protocol tracking clinical manifestations.
  * **Urinary Tract Infection (UTI):** High-frequency acute clinical events.
  * **HIV:** Chronic infectious management profiles.
* **Non-Communicable Diseases (NCDs):**
  * **Type 2 Diabetes Mellitus (T2DM):** Including targeted research sub-cohorts (the **EXPLORE Study** cohort).
  * **Hypertension:** The most common cardiovascular baseline chronic condition.
  * **Asthma & COPD:** Representative respiratory illness baselines.
  * **Cardiovascular Disease:** Chronic heart disease and history of myocardial infarctions.

### 2.2 Indonesian SATUSEHAT Integration
Raw data generated by the simulator is mapped to the Kemenkes SATUSEHAT profile specifications:
* **National Identifiers:** Generates realistic **NIK** (National ID) numbers using Jakarta code prefixes (`3174...`) and custom **IHS Numbers** (e.g. `P10000284...`).
* **Indonesian Names:** Assigns realistic male and female Indonesian names (e.g. *Budi Santoso*, *Siti Lestari*) based on patient gender.
* **Geographical Mapping:** Assigns randomized DKI Jakarta administrative codes (Province `31`, City, District, Village) and postal codes.
* **Healthcare Institution:** Updates all organization and location resource fields to *RSUD Tarakan Jakarta* (Organization ID: `10000004`).

## 3. Configuration & Files

* [synthea.properties](file:///Users/ahmadhidayat/claude-code/projects/FHIR-to-OMOP/step_1_fhir_creation/synthea.properties): Configures Synthea parameters, restricting history to 5 years and enablingObservations, Vitals, Conditions, and RxNorm Medication Request exports.

* [generate_jakarta_patients.sh](file:///Users/ahmadhidayat/claude-code/projects/FHIR-to-OMOP/step_1_fhir_creation/generate_jakarta_patients.sh): Master script coordinating the simulator run and the Python SATUSEHAT profile mapping script.

---

## 4. Execution Commands
To regenerate the cohort, run from the `step_1_fhir_creation` directory:
```bash
# Generate default size cohort (100 patients)
./generate_jakarta_patients.sh

# Or generate a custom cohort size (e.g. 200 patients)
./generate_jakarta_patients.sh 200
```
Outputs are stored as aligned FHIR R4 JSON bundles inside [data/fhir/jakarta_hospital/](file:///Users/ahmadhidayat/claude-code/projects/FHIR-to-OMOP/step_1_fhir_creation/data/fhir/jakarta_hospital/).
